## Affected Pages/Routes

- `/` — 게스트 입장 화면 (닉네임 입력 → 로비 진입)
- `/register` — `@trizn.kr` 회원가입
- `/login` — 등록자 로그인
- `/lobby` — 글로벌 로비 (친구목록, 채팅, 방 리스트, 게임 선택)
- `/game/:game_type/:room_id` — 멀티게임 방
- `/play/:game_type` — 싱글게임 (2048, Minesweeper, Pac-Man)
- `/dm/:friend_id` — 1:1 채팅
- `/admin/login` — Admin 로그인 (`.env` 고정 계정)
- `/admin/users` — 사용자 관리 (검색/필터/ban)
- `/admin/rooms` — 활성 방 모니터 + 강제 종료
- `/admin/chats` — chat_logs 검토
- `/admin/stats` — DAU/게임별 카운트 통계

## Key Interactions to Verify

- 닉네임 입력 → 즉시 입장 (게스트, DB 저장 없음, sessions row 생성)
- 회원가입 폼: `@trizn.kr` 외 도메인 거부 (UI 에러 + 서버 검증 둘 다)
- 친구 추가 버튼 → 상대방 LiveView에 실시간 알림 (Phoenix Presence broadcast)
- 친구 추천 사이드바: 처음 10명 + "더보기" 클릭 시 페이지네이션
- 방 생성: 비번 설정 시 `password_hash` 컬럼 채워짐
- 방 참여: 비번 틀리면 거부 + 시도 카운트
- 호스트 강퇴 버튼 → 대상 즉시 leave + 5분 재참여 차단
- Tetris: 60fps 클라이언트 시뮬, 가비지 라인은 서버 broadcast
- Bomberman: 1명 새로고침 → 3명으로 게임 계속
- 채팅: 글로벌 = `lobby:main` 채널, DM = 양방향 친구만 가능
- Admin 로그인 실패 5회/분 → rate limit (Hammer 라이브러리 등)
- Admin ban → 대상 사용자 모든 active session 무효화 + 즉시 disconnect

## Edge Cases

- 게스트 닉네임 중복 (다른 게스트가 같은 닉네임 사용 중)
- 등록자 닉네임 = 게스트 닉네임 충돌 시 우선순위
- @trizn.kr 대소문자 (대문자 입력 시 정규화)
- 친구 자기 자신 추가 시도
- 친구 요청 받은 상태에서 동일인에게 재요청 (idempotent: 같은 동작 두 번 일어나도 결과 한 번만 적용)
- 방 max_players 초과 입장 시도
- 방 호스트가 disconnect → 호스트 권한 다음 사람에게 이전?
- 호스트 강퇴된 사람의 5분 ban 우회 (다른 닉네임/IP)
- Tetris 게임 중 새로고침 = 즉시 패배 처리 + 결과 저장
- Bomberman 4인 중 마지막 1명 남음 = 자동 승리
- Skribbl 그림 그리는 사람이 disconnect = 다음 차례로 즉시 진행
- 채팅 메시지 길이 초과 (e.g., 2000자+) → 거부
- 채팅 도배 방지 (10초에 5메시지 이상 = throttle)
- chat_logs Mongo 인덱스 누락 시 admin 검색 느림
- 관리자가 자기 자신 ban 시도 (admin은 DB에 없으므로 N/A but 검증)
- `.env` ADMIN_PASSWORD_HASH 빠진 채 부팅 → fail-fast crash
- 동시 가입 (같은 이메일 race) → unique_constraint
- 게임 종료 시 MongoDB 다운 → game_events 손실 vs 큐잉
- 재배포 중 게임 진행 → 사전 안내 배너 5분 전 표출

## Critical Paths (E2E)

1. 게스트 + 등록자 두 브라우저 같은 방 입장 → 게임 → 결과
2. 회원가입 → 로그인 → 친구 추가 → DM 보내기 → DM 받기
3. Tetris 1v1: 두 브라우저 매칭 → 60초 플레이 → 게임오버 → match_results 저장 확인
4. Bomberman 4인: 4 브라우저 입장 → 1명 disconnect → 3명 게임 계속 → 1명 승리
5. 호스트 강퇴 → ban 5분 → 5분 후 재참여 가능
6. Admin 로그인 → 사용자 ban → 대상 즉시 모든 탭에서 logout
7. Admin 잘못된 비번 5회 → 1분 lockout
8. 재배포 시나리오: 로그인 상태 유지(sessions DB-backed), 게임방 wipe, 사용자 안내 배너 표출
9. 새 게임 모듈 추가 시 GameRegistry 자동 인식 (모듈 인터페이스 검증)
10. 신고된 채팅 → admin이 chat_logs에서 검색 → ban 액션 → admin_actions 로그 생성
